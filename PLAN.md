# PLAN: Text-to-Speech Server

## 1. Цели и ограничения
- Поднять локальный HTTP-сервер (`localhost`) для синтеза речи из текста.
- Базовый язык: русский (`ru`), с поддержкой других языков (multi-language).
- Кроссплатформенность: запуск на Windows и Linux (включая WSL).
- Приоритет GPU для инференса; автоматический fallback на CPU.
- Полностью локальное решение: без облачных API, без API-ключей, без платных зависимостей.
- Минимальный API: принять текст -> вернуть аудио-файл/байты.

## 2. Технический стек (предлагаемый)
- Язык: Python 3.11+.
- Web framework: FastAPI + Uvicorn.
- TTS движок: Coqui XTTS v2 (мультиязычный, хорош для ru/en и др.).
- ML runtime: PyTorch (CUDA при наличии, CPU fallback).
- Аудио обработка: `soundfile`/`scipy`.
- Конфиг: `.env` + pydantic settings.
- Модель и инференс: локально на машине, без внешних запросов в интернет во время синтеза.

## 3. Архитектура сервиса
- `app/main.py`: FastAPI приложение и роуты.
- `app/config.py`: чтение конфигурации и флагов.
- `app/tts_engine.py`: инициализация модели, выбор device (`cuda`/`cpu`), синтез.
- `app/schemas.py`: Pydantic-схемы запроса/ответа.
- `app/audio.py`: сохранение/стрим WAV.
- `tests/`: smoke + API тесты.
- `scripts/`: утилиты запуска и проверки окружения.

## 4. API контракт (MVP)
1. `POST /v1/tts`
- Request JSON:
  - `text` (string, required)
  - `language` (string, default `ru`)
  - `voice` (string, optional, имя пресета голоса)
  - `speed` (float, optional)
  - `format` (`wav` на старте)
- Response:
  - `audio/wav` (stream) или JSON с ссылкой на файл (на выбор в реализации).

2. `GET /health`
- Проверка состояния сервера и модели.

3. `GET /ready`
- Готовность инференса (модель загружена, device определён).

4. `GET /v1/voices`
- Список доступных голосов (локальные пресеты/сэмплы), чтобы клиент мог выбрать `voice`.

## 5. GPU/CPU стратегия
- При старте сервиса:
  - проверить `torch.cuda.is_available()`;
  - если `true` -> device=`cuda`;
  - иначе device=`cpu`.
- Логировать выбранный device и причину fallback.
- Добавить env-переключатель `TTS_DEVICE=auto|cuda|cpu`.
- Если принудительно `cuda`, но недоступно -> безопасный переход на CPU + warning.

## 7. Деплой и запуск (локально)
- Windows:
  - `python -m venv .venv`
  - `.\.venv\Scripts\Activate.ps1`
  - `pip install -r requirements.txt`
  - `uvicorn app.main:app --host 127.0.0.1 --port 8000`
- Linux:
  - `python3 -m venv .venv`
  - `source .venv/bin/activate`
  - `pip install -r requirements.txt`
  - `uvicorn app.main:app --host 127.0.0.1 --port 8000`

## 8. Риски и решения
- Большой размер модели и долгий cold start:
  - preload при старте, health/readiness endpoints.
- Проблемы CUDA/драйверов:
  - чёткий fallback на CPU, диагностический endpoint.
- Качество русской речи и скорость:
  - подбор модели/голоса, возможность выбора `voice`.
- Лицензии и офлайн-режим:
  - использовать только open-source модели/библиотеки с разрешённой лицензией для локального использования.

## 9. Критерии готовности MVP
- Сервер стартует на Windows и Linux.
- `POST /v1/tts` стабильно отдаёт корректный WAV для русского текста.
- GPU используется автоматически при доступности.
- При отсутствии GPU сервис продолжает работать на CPU без падений.
- Никаких API-ключей и обязательных внешних платных сервисов.
- Доступно несколько голосов, и их можно выбрать через API.
- Есть краткая инструкция запуска и проверки.

## 10. Что уточнить перед реализацией
- Формат ответа по умолчанию: сразу `audio/wav` или хранить файл и отдавать URL.
- Минимальное число голосов в первой версии (например, 3 или 5).
- Ограничение на длину текста в одном запросе.
- Нужна ли аутентификация (для локального режима обычно нет).
